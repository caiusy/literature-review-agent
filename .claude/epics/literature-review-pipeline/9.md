# Task: 端到端测试 — 用真实关键词跑完全流程

## Description

用一个真实的研究关键词跑完整个流水线：检索→筛选→下载→转换→阅读→综述，验证所有组件协同工作。

## Acceptance Criteria

- [ ] 选择一个真实关键词（如 "efficient LLM fine-tuning" 或 "bev detection"）
- [ ] search-agent 成功检索候选论文
- [ ] 单篇测试时从候选中选择 1 篇开放获取论文
- [ ] PDF 下载成功
- [ ] marker-pdf 转换成功，Markdown 保留全文
- [ ] literature-reviewer 为该论文生成结构化笔记
- [ ] synthesis-agent 生成简要综述/总结
- [ ] 所有中间产物可查看

## Technical Details

测试步骤：
1. 启动 MCP server（确认 opencode.json 配置正确）
2. 在对话中触发 search-agent："搜索 efficient LLM fine-tuning"
3. 查看 data/search-results/filtered.md，确认筛选结果
4. 查看 data/papers/ 和 data/papers-md/，确认下载和转换
5. 触发 literature-reviewer："阅读所有论文"
6. 查看 data/notes/，确认笔记质量
7. 触发 synthesis-agent："生成综述"
8. 查看 outputs/review-overview.md，确认综述质量

记录测试结果到 docs/implementation-log.md。
