# Improve convert_pdf performance with fast extraction mode

## Context

The current default PDF conversion path uses marker-pdf, which has excellent layout fidelity but can take many minutes on first use because it downloads and initializes multiple OCR/layout models. This is too slow for quick testing.

## Description

Add a fast conversion mode that uses lightweight PDF text extraction by default, targeting completion within 1 minute for typical single-paper test runs. Keep marker-pdf as an optional high-fidelity layout mode.

## Acceptance Criteria

- [ ] `convert_pdf` supports `mode=fast` and `mode=layout`
- [ ] default mode is `fast`
- [ ] fast mode uses a lightweight local parser and avoids heavy model initialization
- [ ] existing layout-preserving slow mode remains available
- [ ] docs explain when to use each mode
