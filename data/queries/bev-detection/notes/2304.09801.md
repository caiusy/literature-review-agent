# MetaBEV: Solving Sensor Failures for BEV Detection and Map Segmentation

- Source file: `data/queries/bev-detection/papers-md/2304.09801.md`
- Authors: Chongjian Ge, Junsong Chen, Enze Xie, Zhongdao Wang, Lanqing Hong, Huchuan Lu, Zhenguo Li, Ping Luo
- Year / Venue: 2023, arXiv preprint

## Research question
How can BEV perception for autonomous driving remain accurate for 3D object detection and BEV map segmentation when camera or LiDAR inputs are corrupted or entirely missing, while also supporting joint multi-task learning?

## Method
MetaBEV encodes camera and LiDAR inputs with modality-specific BEV encoders, then initializes dense learnable meta-BEV queries. A BEV-Evolving decoder iteratively updates these queries using cross-modal deformable attention over camera-BEV, LiDAR-BEV, or both, plus self-attention to model query interactions. To reduce task conflict between detection and segmentation, the decoder replaces standard FFN layers with an M2oE module, implemented as routing-based or hard multi-task mixture-of-experts. The model is trained with switched-modality training, randomly using camera-only, LiDAR-only, or fused inputs.

## Data / benchmark
Experiments are conducted on nuScenes, using its 700/150/150 train/val/test split. Tasks are 3D object detection and BEV map segmentation. Robustness is evaluated under six corruption types: Limited Field, Missing Objects, Beam Reduction, View Drop, View Noise, and Obstacle Occlusion, plus Missing Camera and Missing LiDAR settings.

## Key findings
On nuScenes val with full camera+LiDAR input, MetaBEV reaches 68.0 mAP / 71.5 NDS for detection and 70.4 mIoU for map segmentation, setting a new segmentation SOTA reported in the paper. Under Missing Camera, it still achieves 63.6 mAP, 69.2 NDS, and 53.7 mIoU; under Missing LiDAR, 39.0 mAP, 42.6 NDS, and 54.4 mIoU. Compared with BEVFusion in missing-LiDAR conditions, it improves detection by 35.5 NDS points and segmentation by 17.7 mIoU points. In multi-task learning, M2oE improves the shared model to up to 69.8 NDS and 66.9 mIoU.

## Limitations
The paper explicitly notes that deformable attention increases parameters and computational overhead relative to lighter fusion designs. Results are centered on nuScenes; broader cross-dataset validation and real deployment behavior under sensor failures are not explicitly demonstrated.

## Relevance to my topic
This paper is directly relevant to BEV detection because it targets robust BEV fusion under sensor degradation, a core issue for real-world autonomous driving. It is especially useful for work on camera-LiDAR BEV detectors that must tolerate missing modalities while retaining competitive full-modality performance.
