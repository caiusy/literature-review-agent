# QLoRA: Efficient Finetuning of Quantized LLMs

- Source file: `data/papers/2305.14314v1.pdf`

## Research question

1. 能否在极低显存条件下，对超大语言模型进行高质量指令微调。
2. 量化训练能否在显著节省内存的同时，基本保持与 16-bit 全精度微调相当的性能。

## Method

QLoRA 的核心做法是冻结预训练语言模型主干，将其以 4-bit 量化形式存储，只训练 LoRA 适配器参数。

关键技术包括：

1. NF4 (NormalFloat 4-bit)：针对近似正态分布权重设计的 4-bit 量化数据类型。
2. Double Quantization：对量化常数再次量化，进一步节省内存。
3. Paged Optimizers：降低训练时显存峰值。

梯度通过量化后的冻结模型传播，仅更新 LoRA 模块，从而以较低资源完成微调。

## Data / benchmark

实验覆盖多个模型和规模，包括 LLaMA 与 T5 系列，范围从 7B 到 65B。

评测主要包括：

1. 8 个指令微调数据集上的系统比较。
2. Vicuna benchmark 上的对话能力评估。
3. 人工评测与 GPT-4 评测。
4. Guanaco 系列模型的案例分析和失败样例分析。

论文摘要层面未完整展开所有单独数据集名称。

## Key findings

1. QLoRA 能在单张 48GB GPU 上微调 65B 模型。
2. 在性能上基本保留了 16-bit 全精度微调效果。
3. 基于 QLoRA 的 Guanaco 在 Vicuna benchmark 上达到接近 ChatGPT 的表现，文中给出 99.3%。
4. 小而高质量的指令数据集，往往比更大但噪声更多的数据更有效。
5. GPT-4 评测与人工评测有较高一致性。
6. 作者指出现有聊天 benchmark 并不完全可靠。

## Limitations

1. benchmark 的可信度有限，高分不等于真实全面能力。
2. Guanaco 在部分失败案例中仍明显落后于 ChatGPT。
3. 方法主要解决高效微调，不直接解决事实性、安全性、长程推理和鲁棒性问题。
4. 方法效果仍较依赖高质量指令数据。
5. 某些具体任务上的退化边界，需要结合正文实验表进一步核对。

## Relevance to my topic

这篇论文对“大模型高效微调 / 参数高效训练 / 低资源部署”高度相关。

主要价值：

1. 提供了量化 + LoRA 的实用范式。
2. 证明 4-bit 量化不仅能用于推理，也能支持高质量微调。
3. 对有限算力下适配大模型的研究非常重要。
4. 对评估方法也有启发：不能只看单一 benchmark。
